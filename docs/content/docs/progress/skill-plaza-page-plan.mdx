# 技能广场独立页面规划

本文档规划在无限画布里新增一个**独立的技能广场页面**，用于沉淀 AI 漫剧 / AI 短剧生产链路里的专项技能（skill）。

重要形态约定：技能广场是一个独立路由页面（类似现有的提示词库页），**不是**在画布上铺节点。画布只是技能"使用"时的落地目标。

## 背景与对标

对标产品是 B 站 updream（`www.updream.cn`）的技能广场：

- 首页有大输入框「输入创意想法，按 `/` 调用技能开始创作」，下方是「官方精选技能」卡片区 + 「查看全部」入口。
- 独立广场页 `/skills`：「全部 / 热门」tab + 技能搜索框 + 卡片网格。
- 技能卡样式（2026-07-09 实测观察）：
  - 三列网格，每张卡渐变色底（粉 / 蓝 / 橙等，各卡不同）。
  - 左上角：技能 icon + 技能名 + 作者名。
  - 中部：两行描述，超出省略号。
  - 左下角：`使用次数 ⚡ 5.0k`。
  - 右侧：一张竖版封面图。

updream 官方精选技能（AI短剧资产设计师、角色多视图+细节展示、SD2.0 电影级提示词优化、角色图微调器、第一人称视角提示词等）正是本广场首批技能的来源与命名参照。

## 页面设计

### 路由与入口

- 新增路由：`web/src/app/(user)/skills/page.tsx`（与现有 `prompts`、`assets`、`canvas` 同级）。
- 顶部导航新增「技能广场」入口，与提示词库并列。

### 页面结构

```txt
┌──────────────────────────────────────────────┐
│ 技能广场                                [搜索框] │
│ [全部] [热门] [资产设计] [剧本分镜] [提示词导演]   │
│        [场景一致性] [成片质检]                   │
├──────────────────────────────────────────────┤
│ ┌────────┐  ┌────────┐  ┌────────┐           │
│ │ 技能卡  │  │ 技能卡  │  │ 技能卡  │  … 三列网格 │
│ └────────┘  └────────┘  └────────┘           │
└──────────────────────────────────────────────┘
```

- tab 维度：`全部`、`热门`（按使用次数排序）+ 5 个分区筛选。
- 技能卡（对标 updream）：渐变底、icon + 技能名 + 作者、两行描述、使用次数、竖版封面位、状态角标。
- 点击卡片 → 详情抽屉（Drawer）：展示完整字段（解决的问题、输入、输出、必需参考资产、执行步骤、禁止事项、质量标准、上下游技能），底部操作按钮：
  - `在画布中使用`：可运行技能在当前画布插入预置生成流程（提示词节点 + 配置节点 + 生成节点连线）；
  - `复制提示词`：只做提示词 / 只做检查类技能，复制模板文本。

### 与画布的联动

- 可直接运行的技能：通过画布现有的生成流程创建能力（或 canvas-agent 的 `canvas_create_generation_flow` / `canvas_apply_ops`）把技能落成节点组。
- 只做提示词 / 只做检查的技能：产出文本，由用户粘贴或以文本节点插入画布。
- 广场页本身不做生成，不消耗额度。

## 数据结构

第一版用静态数据文件承载，不引入后端：

```ts
// web/src/app/(user)/skills/skills-data.ts
export type SkillZone =
  | "资产设计"
  | "剧本分镜"
  | "提示词导演"
  | "场景一致性"
  | "成片质检";

export type SkillStatus =
  | "可直接运行"
  | "需要人工确认"
  | "只做提示词"
  | "只做检查"
  | "待开发";

export interface SkillCard {
  id: string;
  name: string;           // 技能名
  zone: SkillZone;        // 分区
  stage: string;          // 阶段（卡片标题格式：[阶段] 技能名）
  author: string;         // 作者
  problem: string;        // 解决的问题（卡片两行描述）
  inputs: string[];       // 输入
  outputs: string[];      // 输出
  refAssets: string[];    // 必需参考资产
  steps?: string[];       // 执行步骤
  forbidden: string[];    // 禁止事项
  quality?: string[];     // 质量标准
  upstream: string[];     // 上游技能 id
  downstream: string[];   // 下游技能 id
  status: SkillStatus;
  usageCount?: number;    // 使用次数（v1 可留空或手填）
  cover?: string;         // 竖版封面图路径
  gradient?: string;      // 卡片渐变色
}
```

## 项目生产红线（全部生图技能强制）

1. 必须使用 `images.edit`（图生图 / 参考图编辑）。
2. 必须传入至少一张参考图。
3. 禁止使用 `images.generate` 做纯文生图。
4. 角色相关必须传入角色参考图（三视图、turnaround sheet、官方 Q 版贴纸）。
5. 背景相关必须传入游戏截图或已有 Q 版背景参考。
6. prompt 必须明确要求匹配参考图风格（如 `match the art style of the reference`）。

任何技能无法满足以上条件时，状态必须标记为「只做提示词」或「待开发」，不得直接生产。

红线在页面上的落点：

- 详情抽屉里固定展示「必需参考资产」和「禁止事项」；
- `在画布中使用` 时，生成流程模板默认带参考图输入节点，参考图为空则禁止触发生成（即「参考图强制守门员」——原设计稿引用过但未定义，此处定义为：所有生图类技能流程模板内置的参考图非空校验，不是一张单独技能卡）。

## 首批技能清单（5 分区 24 张卡）

### 1. 资产设计（青绿色系）

| 技能 | 解决的问题 | 状态 |
| --- | --- | --- |
| AI短剧资产设计师 | 剧本需求 → 角色 / 背景 / 道具 / 服装等资产清单与优先级 | 只做提示词 |
| 角色多视图 + 细节展示 | 单角色整理成正 / 侧 / 背 / 表情 / 服装细节的一致性参考 | 可直接运行 |
| 角色 A-pose / 四视图生成 | 生成标准姿态，供动作、表情、分镜、视频首帧复用 | 可直接运行 |
| 角色图微调器 | 局部修正发型 / 服装 / 表情 / 手部问题，不重画不换人 | 可直接运行 |
| Q版角色一致性检查器 | 检查同一角色多图间身份、服装、比例、画风一致性 | 只做检查 |

### 2. 剧本分镜（蓝色系）

| 技能 | 解决的问题 | 状态 |
| --- | --- | --- |
| 自动剧本生成器 | 选题 + 人物关系 + 时长 → 短剧脚本草案 | 只做提示词 |
| 剧本文案拆解师 | 长脚本 → 场景 / 镜头 / 动作 / 情绪 / 台词 / 音效拆解表 | 只做提示词 |
| 精品分镜工坊 | 拆解表 → 可执行分镜（镜号 / 景别 / 机位 / 动作 / 台词） | 需要人工确认 |
| 镜头语言导演 | 普通分镜 → 有镜头感的视频语言（推拉摇移 / 节奏 / 焦点） | 只做提示词 |
| 分镜表格生成器 | 分镜内容 → 统一字段的标准分镜表 | 只做提示词 |

### 3. 提示词导演（紫色系）

| 技能 | 解决的问题 | 状态 |
| --- | --- | --- |
| Seedance 2.0 提示词助手 | 分镜表 → 逐镜头视频提示词 + 负面 / 连续性约束 | 只做提示词 |
| 视频提示词优化师 | 普通提示词 → 更稳定可控少歧义的版本（多模型适用） | 可直接运行 |
| 第一人称视角提示词 | POV 主观镜头提示词，带临场感与视角连续性 | 只做提示词 |
| SD2.0 电影级摄影提示词 | 分镜描述 → 电影感摄影语言（光线 / 景深 / 构图 / 色调） | 只做提示词 |
| 全能提示词合规优化 | 检查提示词过长 / 冲突 / 违反模型或项目规则 | 只做检查 |

### 4. 场景与一致性（橙色系）

| 技能 | 解决的问题 | 状态 |
| --- | --- | --- |
| 截图场景多机位提取器 | 游戏截图 → 场景结构 + 多机位多景别背景生成需求 | 只做提示词 |
| 统一影调 3.0 | 同场戏多镜头统一光线 / 色调 / 时间 / 天气 / 情绪 | 只做提示词 |
| 锚点图多场景生成 | 以锚点图为核心扩展同场景不同景别 / 机位 / 动作 | 可直接运行 |
| 背景 Q版转换器 | 真实游戏截图 → Q 版背景（保结构、守参考） | 可直接运行 |
| 场景连续性审查员 | 检查空间 / 站位 / 镜头方向 / 道具位置连续性 | 只做检查 |

### 5. 成片质检（红色系）

| 技能 | 解决的问题 | 状态 |
| --- | --- | --- |
| 角色漂移检查 | 镜头内 / 跨镜头的脸、服装、发型、体型漂移定位 | 只做检查 |
| 越轴检查 | 连续镜头方向 / 视线 / 运动方向越轴检测 | 只做检查 |
| 镜头衔接检查 | 镜头间生硬、跳跃、缺承接的定位与重排建议 | 只做检查 |
| 台词字幕检查 | 台词 / 字幕 / 配音 / 画面动作匹配检查 | 只做检查 |
| 预告片感优化器 | 普通片段 → 有钩子、节奏、情绪张力的版本 | 只做提示词 |

完整字段（输入 / 输出 / 执行步骤 / 质量标准 / 上下游）见源设计稿：`ai-manga-workflow/docs/infinite-canvas-specialized-skill-plaza.md`，实现时逐卡搬入 `skills-data.ts`。

### 推荐链路（详情抽屉里展示上下游）

```txt
标准生产链路：自动剧本生成器 → 剧本文案拆解师 → 精品分镜工坊
              → 镜头语言导演 → Seedance 2.0 提示词助手 → 成片质检

角色资产链路：AI短剧资产设计师 → 角色多视图 → A-pose/四视图
              → 角色图微调器 → Q版一致性检查器

背景场景链路：截图多机位提取器 → 背景Q版转换器 → 锚点图多场景
              → 统一影调 3.0 → 场景连续性审查员
```

## 最小可用版本（v1）

1. `/skills` 路由页：tab 筛选 + 搜索 + 卡片网格 + 详情抽屉，深色主题，卡片样式对标 updream。
2. 静态数据先做 12 张核心卡：AI短剧资产设计师、角色多视图、A-pose/四视图、Q版一致性检查器、自动剧本生成器、剧本文案拆解师、精品分镜工坊、Seedance 2.0 提示词助手、视频提示词优化师、截图多机位提取器、统一影调 3.0、角色漂移检查。
3. 详情抽屉先只做「复制提示词」；`在画布中使用`（插入生成流程模板）作为 v1.5。
4. 封面图可先用渐变底 + 名称首字，后续再补竖版封面。

## 后续扩展

- 第二批技能：表情包生成助手、动作姿态库、首帧图导演、配音脚本整理、音效提示词、封面图助手、发布切片助手。
- 第三批技能：资产包自动整理、角色关系图、多集连续性检查、爆款节奏拆解、平台审核风险检查。
- 使用次数统计（本地计数即可）、用户自建技能、技能导入导出。

## 上架前检查清单

每个新技能进广场前必须回答：

1. 解决的是不是一个明确的小问题？
2. 输入清楚吗？输出能被下游直接使用吗？
3. 是否依赖参考图？涉及生图是否强制 `images.edit`？
4. 是否违反禁止纯文生图红线？
5. 有质量标准吗？知道自己的上下游吗？
6. 需要人工审核吗？状态标对了吗？
