## 1. 背景与动机

Toonflow 现有方法论源自旧 `ai-short-drama`（`web/src/lib/toonflow/prompts.ts` 第 2 行标注同步日期 2026-07-12）。2026-07-16 用户交付了 `ai-short-drama-plus`，并在 2026-07-16~07-21 用它手工完成了跑刀 S1 v3.17（82.5s 成片）。实战撞出的经验目前只存在于会话日记，未回灌进 Toonflow。

两套方法论在最核心处**直接对立**，不是增量关系：

| 维度 | Toonflow 现状（旧线） | plus |
|:---|:---|:---|
| 视频提示词 | 九宫格 1:1、原生多镜头直出、**禁止首尾帧** | Module3 故事板**即首帧** + Module4 六段式 |
| 故事板 | 黑白粗线稿（`buildStoryboardPagePrompt`） | 照相级、16:9、3+2/3+3/4+4 矩阵、三层空间语法 |
| 首帧 | 独立节点，线稿上色（`buildKeyframesPrompt`） | 无此步骤，故事板本身就是首帧 |
| 用词 | 自由描述 | 封闭词库逐字选取，禁自创 |
| 缺失 | — | P0 创意层 / ST 13 色板 / P7 七项质检 / 分镜决策锁定表 |

因此 `storyboard-page` 与 `keyframes` 两个节点的语义必然变化，"只换文字不动节点"不可行。

## 3. 三块划分

| 块 | 范围 | 验收标准 |
|:---|:---|:---|
| **第一块 · 文本决策层** | P0 创意（选修）→ P1 剧本 → P2 资产 → P3 分镜决策 + P4 封闭词库；缝合同签订；台词类型标注 | 锁定表出得对、分镜表带 Layer 1 前件、镜头合同逐字选词、检查器能报出问题并给修改方案 |
| **第二块 · 图像层** | P4.5 俯视调度图 → P5 Module3 照相级故事板 + ST 色板锚定；缝合同画进故事板 | 照相级故事板能直接当首帧用；上段末格是动作中间态、下段首格接后半 |
| **第三块 · 视频与交付层** | Module4 六段式 → P6 生成 → P7 七项质检 → P8 交付；`audio-mix` 激活；`seam-check` 改为核对上游合同；剪辑手法十条 | 一集端到端成片；台词轨对得上口型；缝无跳切 |

本文档详述**第一块**，二三块在各自 spec 展开。

## 4. 第一块设计

### 4.2 封闭词库：内置模块，不做节点

`05-closed-libraries.md` 是 34891 字节静态资源（构图 8 / 布光 10 / 运镜 8 / 景别 L0-L5 / 表演 L1-L5 / 调色 22 / 空镜 A-E / 导演风格 9 / 开场钩子 4 类）。

**不做成画布节点**，理由：静态资源做成节点意味着每个项目都要"生成"一遍词库，无意义；且 `MAX_CONTEXT_CHARS = 8000` 塞不下全库。

**做法**：新增 `web/src/lib/toonflow/closed-libraries.ts` 内置常量模块，按环节注入相关子集（如镜头合同只注入构图/布光/运镜/景别四类），并提供 `isInLibrary(category, term)` 供校验器使用。

### 4.3 接缝贯穿：缝合同在第一块签

**核心认识：怎么分段，本身就是缝的决策。** 段边界定下，缝的位置就定死了。

分段规则改写（`storyboard-table`）：
- 以贴满 15s 打包连续因果画格为目标，连续动作优先整段塞进一次生成（段内镜头切换由模型原生完成，天然无缝）
- **真缝只许落在场景/时间转换处**（连续性需求最低点）

缝合同四行（每个缝一份，存进 `directing-lock` 的段间字段，供第二块画进故事板、第三块核对）：

```
【缝合同 · 上段|本段】
上段末拍 = {动作做到中间态截止}
本段首格 = {同一动作的后半段，禁止重新建立空间}
景别/动机 = {跳≥2档 / 正反打（严守180°轴线）/ POV 运镜动机 / 时间省略}
声音桥   = {J-cut：本段台词或音效提前 0.2-0.5s ｜ L-cut：上段声音拖尾 0.3-0.6s}
```

> **原拟的第五行「音频可分点」已作废** —— 该约束来自跑刀实战（"墩柜声与'久等了'人声同帧不可分"），根因是台词被烧进视频。D8 台词剥离后此约束不再存在。

### 4.4 台词剥离与口型（D8）

**视频层只生成环境音效，台词独立成轨后期剪入。** 口型靠 prompt 控制，不靠独立 API。

`AudioLine` 新增 `type` 字段：

| 类型 | 视频 prompt 要求 | 配音轨 |
|:---|:---|:---|
| `dialogue` 出口对白 | 该角色做口型，写明所说台词文本（控口型，但不注入音频） | TTS 角色音色 |
| `os` 旁白/内心 | **全段全员闭口**，任何人不得对该内容做口型 | TTS 旁白音色 |
| `sfx` 音效 | 与口型无关，正常写进画面 | 视频自带 |

规则源：`video-voice-lipsync.md`（manga-drama 线，plus 未吸收，本次一并搬入）。

第一块落地范围：
1. `storyboard-table` 的 `line` 字段语义变为「配音脚本 + 口型控制文本」，并标注类型
2. `shot-contract` 新增口型要求（本镜谁张嘴、谁闭口），避免"旁白在响、画面里三人同时动嘴"
3. 实际关闭人声生成属第三块（cano 无 `generate_audio` 开关，只能靠 prompt，**须实测验证**）

### 4.5 检查器：提示 + 一键修改方案（D4）

生成后自动运行，结果显示在节点上，**不阻断**，并给出定点修建议：

| 检查项 | 标准 |
|:---|:---|
| 运镜多样性 | ≥2 种 |
| 景别跨度 | ≥3 档（含极值 L0/L5） |
| 角度多样性 | ≥2 种 |
| 构图多样性 | ≥2 种 |
| 同构图连续 | <3 段 |
| 居中对称占比 | <50% |
| 封闭词库合规 | 选词须在库内 |
| 格子数一致 | 分镜表行数 = 故事板格子数（解 TODOS 遗留） |

> **2026-07-21 实施期补正**（初稿漏项，以本节为准）：

**判定粒度**：运镜 / 景别 / 角度是**镜级**（段内按 shotNo 判），构图是**段级**（B 表逐段锁）。初稿此表按镜头描述构图，与源文件 `04-directing-decisions.md:139`「B 表全程至少使用 2 种不同构图策略 / 同构图连续 ≥3 **段**」不符。

**初稿漏掉的三条镜级子规则**（源：`04-directing-decisions.md` §5 + D5.1）：

| 检查项 | 标准 |
|:---|:---|
| 同种运镜连续 | <3 镜 |
| 平视连续 | ≤2 镜 |
| 连续同景别 | <3 镜 |

景别跨度另须**含至少 1 个极值 L0/L5**。D5.1「镜数≥5→≥3 种景别」被无条件的「≥3 档」覆盖，不单独实现。

**短段保护**：段内可解析镜头数 <3 时不判景别跨度、<2 时不判运镜/角度多样性，一律标 `unknown`——物理上凑不出，判 fail 会冤枉 plus 分段规则下合法的短段。连续复用类判定不受此限。

**「居中对称」的真实含义**：构图 8 策略（权力压迫/负空间吞噬/对角线驱动/前景切割/边缘出画/倾斜失衡/尺度反差/纵深分层）本身就是一套**反居中策略集**，库内没有任何词条字面意为「居中」或「对称」。故源文件「禁止全程主体居中对称」指的是**没从 8 策略里选、退化回生成模型默认的居中构图**。判定条件取或：`compositionPrimary` 不在构图 8 策略内，或字面含「对称」/「中心」。照字面去库里找词会导致本检查永远空转。

**一键修改方案**：检查器发现问题后调模型生成定点修补丁，只改不达标的那几镜、其余字段原样不动（非整表重生成）。UI 呈现建议改法 + 理由，支持「全部应用 / 部分应用 / 忽略」。

### 4.6 级联

`directing-lock` 与 `continuity-table` 是全局节点，改动会让所有段的 `storyboard-table` / `shot-contract` / `action-contract` 标 `stale`。这符合 plus 铁律 2 的意图：锁定表本就不该反复改。

### 4.7 双侧同步（实施必读）

方法论红线在 `web/src/lib/toonflow/prompts.ts`（`AGENT_METHODOLOGY_BRIEF` + `STAGE_METHODOLOGY_REDLINES`）与 `canvas-agent/src/config.ts` 中**逐字双份**，由 `__tests__/agent-brief-sync.test.ts` 锁定一致性。改方法论必须两侧同步，否则测试红。

### 4.8 兼容性

新增 schema 字段一律 `optional`，旧画布（如跑刀）能打开不崩，但内容属旧方法论产物，需按新方法论重生成才有意义。

## 5. 风险

| 风险 | 说明 | 处置 |
|:---|:---|:---|
| cano 未验证 plus 实证 | ST 色板锚定、`autoCompliance` 过审等结论均在 LibTV Seedance 上得出 | 第二/三块实测；不平移就退回记录实际表现 |
| cano 无 `generate_audio` | 关不掉人声生成，只能靠 prompt | 第三块实测；若 prompt 无效则改用 Seedance 渠道或后期剥离人声 |
| 口型回避与情绪表演冲突 | 说话镜头若一律不给正脸，爆点镜表演受限 | 第三块实测后再定；本块只标注口型要求，不限制构图 |
| 一键修改可能引入新的不达标 | 定点修改了几镜后其他维度可能反而失衡 | 应用后重跑检查器 |
