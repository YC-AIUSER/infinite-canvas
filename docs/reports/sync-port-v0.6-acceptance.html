过程中处理的环境问题

Windows 下 npm 原生模块三件套（已解决，记入长期记忆）

上游 lockfile 缺 Windows 原生依赖（npm 官方 bug #4828），dev server 连环报 Cannot find module。解法是三个原生包一条命令、版本精确匹配地补装（分开装会互相清掉，版本不匹配照样报错）：

npm install @rollup/rollup-win32-x64-msvc@4.62.2 lightningcss-win32-x64-msvc@1.32.0 @tailwindcss/oxide-win32-x64-msvc@4.2.4 --no-save --legacy-peer-deps

另：该仓库装依赖一律要加 --legacy-peer-deps（上游自带的 antd 6 与 pro-components beta 的 peer 冲突）。这些已存入项目记忆，下次直接用。

残留进程占端口（已清理）

旧 Next dev server 的 node 子进程残留占用 3000 端口，已定位并终止，Vite 现运行在 http://localhost:3000。
