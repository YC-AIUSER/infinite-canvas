Toonflow 生产画布 — 全流程方法论生成能力

把一键生成背后的黑盒摊开成 14 个方法论节点,逐节点生成、七态流转、任意层修改后向下重生成。产品命题:从模板复读机到自己的片子——用户不是获得控制权,是获得一个不让他犯错的流程。

01

问题与证据

  • 真问题 = 支配度 — 粗版一键生成的片子能发能卖,但客户"只能生成跟我模板类似的东西,无法根据自己的想法来做细微调整"(创始人转述客户原话,全场最锋利的一句)。
  • 付费证据 — 有具体的人愿付费买培训/代运营(服务证据,非工具订阅证据;工具本身零直接付费验证,见 P4)。
  • 外部佐证 — 全网对一键生成的一致抱怨:角色漂移/崩脸/每镜重摇 15-20 次;行业共识"瓶颈是控制不是生成"。
  • 竞争现实 — 节点画布已是拥挤品类(PixVerse Canvas、Flora、Opone、火山剧创 99 元/年);功能对齐必输,方法论编码是唯一胜负手。

04

核心机制

七态状态机(继承现有七态改造) 
empty → generating → review → approved
          ↓ 失败        ↑ 重生成/编辑退回      ↓ 直接或间接上游出新版本
        failed(重试)                        stale
skipped:用户显式跳过,下游视为透明;可取消回 empty
  • approved 后编辑:保存即 version+1、保持 approved(编辑者即验收者),触发传播。
  • 失效传播:触发事件=新版本产生(生成完成/编辑保存),每版本传播一次;验收(review→approved)不触发;BFS 每跳带版本守卫(快照已含该版本者跳过)。
  • 级联重生成:拓扑序逐节点,中间节点自动进 review 并作为下游输入,链尾「全链通过」批量验收。
  • 结构化节点(#1/#12/#14):empty→review(编辑)→approved(保存,触发传播);改 #1 全图 stale 是预期行为,可逐个"沿用旧产出"豁免。
段建模与生命周期

60s≈4 段,分段决策落在分镜表(#5,按换场景/换时间自动划段,review 可调)。#8/#9/#11 每段一实例,复用现有 batch 子节点机制。段以稳定 segmentId 为键防重排错绑;#5 新版本 approved 时:存活段标 stale、新段建 empty 实例、消失段归档(保留产物,用户确认后删)。

nodeOutput schema(阶段 1 首个交付物) 
type NodeOutput = {
  nodeId, kind, version,            // 版本历史保留
  status: empty|generating|review|approved|failed|stale|skipped,
  segmentIndex?,                    // 段实例节点必填
  payload: { text? table? imageKeys? videoKeys? audioKeys? },
  upstreamVersions: Record<nodeId, version>,   // stale 判定快照
  error?, generatedAt }

媒体走现有 file-storage(多版本=多 storageKey);文本历史以 NodeOutput[] 存节点 metadata。

07

风险与悬置问题

  • P3 无退路(风险接受,无缓解) — Codex 的可证伪指标(修改成功率≥70%/单镜重摇≤3)与升级路径被创始人否决;若合同层控制不足以支撑"改动作落点"类需求,将在阶段 3 撞墙。由创始人明示承担。
  • 验证后移 — 2-4 周无外部反馈;缓解:The Assignment 先行 + P4 的 14 天时限与回退路径。
  • 付费证据与产品形态错位 — 付费意向在服务,产品是工具;P4 验证失败即回退"工具+陪跑"。
  • Open Q1(阶段 3 前必须关闭) — 部署平台 + 外部用户 key 方案(自带 or 代理计费;代理有滥用/成本失控风险);CORS 转发不可复用(现有 /relay/ 是 Vite dev 插件),需新建 serverless/edge 函数。
  • 其余悬置 — 成本模型(Q2)、粗版对接(Q3,C 期)、词表维护(Q4,二期)、拼接导出(Q5,二期)、点位图交互(Q6,后置)。
08

The Assignment(本周,不写代码)

把画布静态版拿给三个人看——想付费买培训的那位客户 + 两位粗版一键生成的客户,只问一个问题:

"你上次用一键生成,想改但改不了的,是哪一层——剧情/剧本?分镜/镜头顺序?画面/首帧?还是视频动作?"

三个答案原话记下来,直接决定节点接生成的先后顺序。如果三个人都说"没想改过什么"——P1 需要重审,先别开工。
